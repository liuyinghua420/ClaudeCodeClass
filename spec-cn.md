# 设计文档：面向零基础成年人的 Claude Code 课程

由 /office-hours 生成于 2026-04-14
分支：unknown
仓库：ClaudeClass（本地，无 git）
状态：已批准
模式：Builder

## 问题陈述

用 8 次面授课程（每次 2 小时）教零基础成年人高效使用 Claude Code。学员背景多元：有的是想在工作中用 AI 的职场人士，有的是想学做东西的新手，有的只是好奇。到第 8 次课结束时，每位学员都应该用 Claude Code 做出一个真实的项目。

## 这门课的独特之处

这不是又一门"学编程"的课。这是一门"学会指挥 AI 智能体"的课。核心认知转变是：你不是在学一门编程语言，你是在学习管理一个速度极快的初级开发者——它能阅读你的整个代码库、做出决策并执行。这个重新定义让非技术人员也能驾驭 Claude Code，这是传统编程课从未做到的。

市场空白：所有 vibe coding 课程都在教"写提示词 → 得到代码"。没有人在教智能体管理的思维模型（上下文管理、迭代优化、评估 AI 输出、知道什么时候该接受什么时候该反驳）。这才是真正的技能。

## 约束条件

- 8 次课 x 每次 2 小时，面授，学员自带笔记本电脑
- **班级规模：** 10-15 名学员，1 名老师 + 2 名助教（每 5-7 名学员配 1 名助教）
- 学员：零基础，学习动机各异，中文母语成年人
- **语言：** 中文授课。工具界面（终端、Claude Code）保留英文。讲义和幻灯片双语（中文讲解，英文命令/代码）。
- **休息安排：** 每节课约第 55 分钟休息 10 分钟。所有课时安排已考虑此休息时间。
- 第 8 次课前必须做出一个实际可展示的项目
- 环境搭建必须万无一失（这是新手课程最容易失败的环节）
- 老师亲自授课，非录播/异步

### 技术范围

所有项目均为客户端 Web 应用（HTML/CSS/JavaScript），在浏览器中打开即可。无需服务器、数据库或后端。这保证了即时反馈循环：保存文件 → 刷新浏览器 → 看到结果。

### 学员前提条件

- macOS 10.15+ 或 Windows 10+ 的笔记本电脑（不支持平板、Chromebook）
- 拥有机器的管理员/安装权限（无需 IT 审批即可安装软件）
- 稳定的网络连接（教室 Wi-Fi 或个人热点）
- **不需要任何编程经验**

### API Key 策略

老师提供共享 API Key，供课堂使用。学员不需要注册 Anthropic 账号或绑定支付方式。这消除了新手最大的摩擦点。

- **默认模型：** Claude Sonnet（速度快、价格低、满足所有学员练习需求）
- **老师演示用：** Claude Opus（"哇"时刻和现场演示用更高质量的输出）
- **费用预估：** 使用 Sonnet 模型，8 次课 10-15 名学员约 $150-250。预算 $300 留余量。
- **并发处理：** 准备 2-3 个 API Key，分配给不同学员组（Key A 给 1-5 号学员，Key B 给 6-10 号，Key C 给 11-15 号）。这避免 15 名学员同时发请求时触发单 Key 速率限制。课前测试并发上限。
- **课后：** 想继续使用的学员会收到注册个人账号的指南。

### 助教培训

首次课前对助教进行培训（30 分钟）：
- 逐步走完安装指南
- 熟悉常见失败场景：Node.js 版本冲突、PATH 环境变量问题、Windows 权限报错
- 核心原则：帮学员解决卡点，不要替他们做。引导他们自己输入命令，而不是你来输入。
- 分区负责：每位助教覆盖教室特定区域的 5-7 名学员

## 核心前提

1. **项目优先，而非理论优先。** 思维模型从实践中自然产生。没有即时动手环节的理论课会很快失去混合群体的注意力。
2. **环境搭建值得专门花时间。** 让零基础的人装好 Node.js + Claude Code + 终端就是一堂完整的课。
3. **学员学的是指挥 AI，而非从零写代码。** 核心技能是：描述需求、迭代优化、用 AI 调试、评估输出结果。
4. **8 次课留出足够空间。** 零基础 + 项目成果 + 混合群体需要时间来追赶和个性化调整。

## 备选方案

### 方案 A：线性构建
在第 4-8 次课中逐步构建一个项目。进度清晰但对混合群体不够灵活。如果项目不合某人口味，他们就被困在里面了。

### 方案 B：里程碑冲刺（已选择）
引导式 → 半引导式 → 独立式。第 1 次课制造"哇"时刻。第 3-4 次课构建引导式小项目。第 5-6 次课提供选择。第 7-8 次课是个人项目加辅导。逐步建立真正的自主能力。

### 方案 C：案例驱动
每次课是一个独立案例。产出 4-5 个小项目的作品集。准备工作最多，但每节课独立成章。风险：学员会觉得自己在做练习，而不是在做"自己的东西"。

## 推荐方案：里程碑冲刺

### 脚手架层级

- **引导式**（第 3-4 次课）：老师逐步带领。每 10-15 分钟一个检查点（"到这里你应该看到 X"）。老师先演示每个步骤，然后学员跟做。所有人做同一类型项目，共享检查点，但在功能上可以个性化。
- **半引导式**（第 5-6 次课）：学员从项目模板出发独立完成。模板包含：项目说明、建议的构建顺序和 3-4 个里程碑检查点。老师巡场，按需帮助，不主动干预。
- **独立式**（第 7-8 次课）：学员构思、确定范围并构建自己的项目。老师按需辅导。无逐步指南。

**课时说明：** 下文每个"第一小时"约 55 分钟教学时间。中间休息 10 分钟。每节课实际教学时间：110 分钟。

### 第 1 次课："哇"时刻 + 环境搭建（2 小时）

**第一小时：制造"哇"时刻（先做这个，在任何安装之前）**
- 老师现场演示 Claude Code：从听众中征集一个想法，10 分钟内做出一个可用的网页
- **想法引导：** 征集想法后，选择/调整一个适合演示的：单页、静态、视觉化的成果（比如一张生日贺卡、一个个人主页、一个餐厅菜单）。拒绝任何需要后端或登录的想法。
- 学员看到完整循环：描述你想要什么 → 看 AI 构建 → 在浏览器中看到结果
- 这就是"我也想做到"的时刻。后面的一切都是为了达到这个目标的脚手架。
- **备用方案：** 老师准备 2-3 个预先测试过的演示想法。如果 API 出问题，播放预录的同流程屏幕录像。

**第二小时：开始环境搭建**
- 预先准备好的安装指南（打印的双语讲义 + 屏幕投影）
- 安装清单：终端基础 → Node.js → Claude Code → 配置共享 API Key
- **关键：** 2 名助教在教室走动帮助。预配置的 USB 驱动器作为主要备份。
- **USB 内容：** Node.js LTS 安装包（macOS + Windows）、Claude Code npm 包离线安装文件、安装指南 PDF、速查卡 PDF。
- 目标：让尽可能多的学员完成安装
- 提前完成的学员：试用速查卡上的 3 个命令

**备用方案：** 无法本地安装的学员立即使用 claude.ai/code（网页版）。确保没有人在第 1 次课结束时还没跟 Claude 交互过。未完成安装的学员在第 2 次课前 20 分钟补完（内置缓冲时间）。

### 第 2 次课：安装缓冲 + AI 思维模型 + 初次动手（2 小时）

**前 20 分钟：安装完成缓冲**
- 第 1 次课未完成安装的学员现在补完（助教协助）
- 已完成安装的学员：自由探索 Claude Code，试用速查卡上的命令

**接下来 40 分钟：如何理解 AI**
- 思维模型 1：Claude Code 是一个初级开发者，你是它的经理。你给方向，它执行，你审查。
- 思维模型 2：上下文决定一切。AI 只能处理它能看到的内容。演示项目结构、文件命名和清晰指令如何改变输出质量。
- 思维模型 3：迭代，而非追求完美。第一次输出是草稿。核心技能是知道如何把它推向你想要的方向。
- 演示：同一个提示词，差的上下文 vs. 好的上下文。展示巨大差异。

**第二小时：引导式练习**
- 学员练习核心循环：描述 → 生成 → 评估 → 优化
- 练习 1：用 Claude Code 做一个个人介绍页面（HTML）
- 练习 2：修改它。换颜色、加一个区块、修一个看起来不对的地方
- 核心技能：阅读 Claude 的输出，判断"对，就是这样"还是"不对，改这里"

### 第 3 次课：小项目 1 —— 引导式构建，上半场（2 小时）

**项目：个人日程规划器（所有人做同一类型的工具）**
- 所有人都做日程规划器，但可以个性化：有人加入饮食记录，有人加入运动追踪，有人加入工作任务
- 同一类型工具 = 同样的检查点对所有人有效。个性化保持趣味性。
- 第 3 次课聚焦：项目结构、向 Claude Code 描述需求、构建核心功能

**第一小时：需求思维**
- 如何把一个大想法拆分成小的、可构建的部分
- 练习：用日常语言写 5 条描述，说明工具应该做什么
- 把它喂给 Claude Code。看看会发生什么。评估。

**第二小时：构建核心功能**
- 学员按步骤跟做引导式构建
- 老师在教室走动，帮助解决卡点
- 关键教学时刻：当 Claude Code 做了意料之外的事情时，如何纠正方向

### 第 4 次课：小项目 1 —— 引导式构建，下半场（2 小时）

**第一小时：添加功能**
- 继续构建：给工具添加 2-3 个功能
- 教学：如何给 Claude Code 提供现有代码的上下文（"先读这个文件，然后添加..."）
- 练习：多轮对话交互，在已有工作基础上继续构建

**第二小时：调试 + 打磨**
- 引入 AI 辅助调试："这个不工作，我期望的是这样，实际发生的是这样"
- 学员修复项目中的问题
- 打磨：让它看起来更好，添加收尾细节
- **里程碑：每个人都有了一个自己亲手做的可用工具**

### 第 5 次课：小项目 2 —— 半引导式构建，上半场（2 小时）

**学员从 3-4 个项目模板中选择：**
- **选项 A：小型商业着陆页** —— 一个带有主视觉区、服务列表、联系表单和响应式设计的单页网站。成品：一个专业的页面，可以实际使用或送给别人。起始脚手架：包含说明和示例截图的项目文件夹。
- **选项 B：个人作品集/兴趣展示** —— 一个多页网站，展示学员的兴趣、作品或爱好。成品：可以分享到微信的东西。起始脚手架：带有占位区块和示例内容的模板。
- **选项 C：简单互动游戏** —— 问答游戏、记忆翻牌游戏或猜数字游戏。成品：可以给朋友们玩的有趣东西。起始脚手架：在说明中描述了游戏循环结构的基础 HTML 页面。
- **选项 D：实用工具** —— 单位转换器、倒计时器、待办清单或带点花样的计算器。成品：一个做好一件事的单页 Web 应用。起始脚手架：包含清晰输入/输出规格的项目说明。

**第一小时：规划你的项目**
- 学员选择模板，定制需求
- 练习：为 Claude Code 写好项目简报
- 老师审阅每位学员的简报，提出改进建议

**第二小时：有脚手架的独立构建**
- 学员以模板为起点进行构建
- 老师提供检查点："到第 30 分钟，你应该让 X 功能工作起来"
- 将进度较慢的学员与较快的学员配对

### 第 6 次课：小项目 2 —— 半引导式构建，下半场（2 小时）

**第一小时：进阶技巧**
- 教学：Claude Code 的计划模式（Plan Mode）—— 在写代码之前，先让 Claude 规划方案。使用 `/plan` 或描述你想要的并说"先规划再构建"。这防止 Claude 在较大项目上一头扎进错误方向。
- 教学：如何处理更大的项目（文件组织、上下文管理）
- 学员将这些技巧应用到自己的项目中

**第二小时：完成 + 分享**
- 完成小项目 2
- 快速展示：每位学员向邻座展示自己的项目（每人 2-3 分钟）
- **布置作业：** "带着一个个人项目的想法来第 7 次课。"现在发放项目创意菜单，让学员这周可以浏览思考。
- **里程碑：学员做出了自己选择的东西，而不仅仅是跟着做**

### 第 7 次课：个人项目 —— 你的想法（2 小时）

**第一小时：创意工坊**
- 每位学员写下："我生活/工作中有什么事情可以通过软件变得更简单"
- **提供项目创意菜单：** 10 个预先确定范围的项目想法，学员可以直接采用或改编。例如：生日提醒页面、餐厅菜单网站、运动追踪器、会议笔记整理器、课程表查看器。有自己想法的学员可以用自己的；菜单消除了面对空白页的困难。
- **第 6 次课的作业：** 学员应该带着项目想法来第 7 次课（第 6 次课末布置）。第 7 次课第一小时是优化想法，而非从零构思。
- 老师帮助将想法精炼到可构建的范围（这是最难的教学时刻）
- 关键规则：如果 5 条描述写不完，那就太大了。缩小范围。老师有否决权。
- 学员写项目简报

**第二小时：构建冲刺**
- 学员构建自己的个人项目
- 老师逐个辅导，在座位间走动
- 重点：帮助解决卡点，而非代做。让学员有效地挣扎。

### 第 8 次课：完成 + 展示会（2 小时）

**第一小时：完成 + 打磨**
- 个人项目的最后构建时间
- 帮助学员准备 2 分钟演示（做了什么、怎么做的、学到了什么）
- **不要求部署。** 学员从本地机器演示。课后指南涵盖部署方法（GitHub Pages、Vercel），供想在线分享项目的学员参考。

**第二小时：演示日**
- 每位学员展示自己的项目（每人 2 分钟）
- 庆祝大家的成果
- 老师提供"接下来怎么走"的资源
- 可选：建立班级微信群用于持续学习

## 成功标准

- 第 1 次课结束时 95%+ 的学员能执行 Claude Code 命令（本地或网页版备用方案）。剩余学员在第 2 次课前完成本地安装。
- 第 4 次课结束时 90%+ 的学员有一个可工作的小项目 1
- 第 8 次课时 80%+ 的学员有一个可演示的个人项目（完成或有实质性进展）
- 学员在课后反馈中表示"我可以自己做了"
- 至少 3 名学员在课后继续独立使用 Claude Code

## 分发计划

课程材料（幻灯片、讲义、项目模板、安装指南）存储在 ClaudeClass 仓库中。老师面授。材料初期不公开分发，但结构化程度足以在课程成功后分享或授权。

## 下一步行动

1. **立即：** 创建第 1 次课安装指南，找 2-3 个零基础朋友测试
2. **第 1 周：** 制作第 1 次课"哇"时刻的演示脚本，练到流畅
3. **第 2 周：** 端到端设计小项目 1（引导式构建），包括所有检查点
4. **第 3 周：** 创建小项目 2 的 3-4 个项目模板（第 5-6 次课用）
5. **开课前：** 找 3-5 人的测试小组，试讲第 1-2 次课
6. **开课前：** 验证网页版备用方案（claude.ai/code 或替代品）能支持核心循环。如不能，准备一台预装 Claude Code 的共享备用电脑。

## 评审意见

以下小问题在评审中被标记，但属于可接受的权衡：
- 课时安排可以更精确到分钟级别，但那种细节留给老师的备课笔记更合适。
- 末尾的"我注意到的"部分是设计过程中的元评论。留作个人参考，分享给助教前删除。

## 需要准备的补充材料

- 打印安装指南（覆膜，可重复使用）
- 速查卡：20 个最常用的 Claude Code 命令/模式
- 项目简报模板（填空格式，第 5-8 次课用）
- "当 Claude 给你错误结果时怎么办"故障排除指南
- 课后资源清单（视频、文档、社区）

## 风险缓解

| 风险 | 缓解措施 |
|------|----------|
| 部分学员安装失败 | 网页版备用（claude.ai/code）、预配置 USB 驱动器、2 名助教、第 2 次课有 20 分钟安装缓冲 |
| 学员担心 API 费用 | 老师提供共享 API Key（总计约 $150-250，预算 $300）。2-3 个 Key 分配给不同学员组以避免速率限制。 |
| 混合群体技能差距大 | 结对编程（快+慢配对）、差异化检查点、多级项目难度 |
| 个人项目范围膨胀（第 7 次课） | 硬规则：最多 5 条描述。提供项目创意菜单。老师有否决权。 |

## 我注意到你的思维方式

- 你说的是"0基础的成年人"，不是"有点技术基础的初学者"。你在认真对待最难的教学问题，没有降低标准。
