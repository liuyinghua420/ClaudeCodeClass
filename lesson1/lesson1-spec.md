## 课程信息

| 项目 | 内容 |
|------|------|
| 课次 | 第 1 次课（共 8 次） |
| 主题 | "哇"时刻 + 环境搭建 |

## 学员在本课结束时应该达到的状态

1. **内心被点燃**：亲眼看到 AI 在 10 分钟内把一个想法变成可用网页，产生"我也想做到"的强烈欲望
2. **环境就绪**：95%+ 学员的电脑上能跑通 Claude Code 命令（或使用网页版备用方案）
3. **完成首次交互**：每位学员都至少跟 Claude Code 对话过一次，看到过它的输出

## 详细教学内容

### 第一部分：开场（0:00 - 0:10）

#### 自我介绍（5 分钟）

**老师说：**

> 大家好，欢迎来到 Claude Code 实战课程。我是 [名字]。
>
> 先说结论：8 次课之后，你们每个人都会做出一个自己的软件项目。不需要任何编程基础。
>
> 这怎么可能？因为我们不是在学编程。我们在学一件更厉害的事 —— 指挥 AI 帮你编程。
>
> 想象一下：你身边有一个速度极快的程序员，24 小时待命。你只需要用中文告诉它你想要什么，它就帮你做出来。这就是 Claude Code。
>
> 不信？让我演示给你们看。

**要点：**
- 控制在 5 分钟内，不要拖长
- 关键信息：8 次课 → 做出自己的项目 → 不需要编程基础
- 立刻引向演示，不要在开场讲太多理论

#### 铺垫（5 分钟）

**老师说：**

> Claude Code 是 Anthropic 公司做的一个 AI 编程助手。它跑在你电脑的终端里。
>
> 你用中文（或英文）告诉它想做什么，它就帮你写代码、创建文件、运行程序。
>
> 它不是一个聊天机器人 —— 它是一个能直接在你电脑上干活的 AI 智能体。
>
> 现在，给我一个想法，我来演示。

**过渡到演示：** 直接向学员征集想法。

---

### 第二部分："哇"时刻演示（0:10 - 0:30）

#### 演示流程（10 分钟）

**第 1 步：征集想法（2 分钟）**

**老师说：**

> 有没有人想要一个网页？什么都行。一个生日贺卡？一个餐厅菜单？一个个人主页？你们说，我来做。

**想法引导规则：**
- 接受：单页网站、贺卡、菜单、个人介绍页、活动邀请函、产品展示页
- 拒绝（温和地）：需要登录的、需要数据库的、需要支付功能的 → "这个很好！不过我们先从简单的开始，这个留到后面的课做"
- 如果没人主动说，用预备方案："那我来做一个 —— 帮我们班做一个课程介绍页怎么样？"

**第 2 步：现场构建（6-7 分钟）**

打开终端，确保投影清晰可见。

```bash
# 创建项目文件夹
mkdir demo && cd demo

# 启动 Claude Code
claude
```

然后用自然语言描述需求。例如（假设学员选了"餐厅菜单"）：

```
帮我做一个中餐馆的菜单网页。餐厅叫"好味道"。
要有：
- 漂亮的标题和餐厅名字
- 分类：凉菜、热菜、汤、主食
- 每个菜有名字和价格
- 页面要好看，用暖色调
```

**老师边做边解说（关键！）：**

> 看，我现在在跟 Claude 说话。我用的是中文，很随意的描述。
>
> 它现在在想...（等待时）看到了吗？它在分析我的需求。
>
> 现在它开始写代码了。注意，我完全不需要知道这些代码是什么意思。
>
> ...好，它说做完了。让我们打开看看。

```bash
# 在浏览器中打开
open index.html    # macOS
# 或 start index.html  # Windows
```

**第 3 步：展示结果（1 分钟）**

切换到浏览器，展示网页。让学员看到一个真实的、漂亮的网页。

**老师说：**

> 从我输入描述到现在，大概 2 分钟。一个完整的网页就做好了。
>
> 8 次课之后，你们每个人都能做到这个，而且做的是你自己想要的东西。

**预期反应：** 教室应该有明显的惊叹/讨论声。给 30 秒让学员消化。

#### 拆解演示（10 分钟）

**老师说：**

> 刚才发生了什么？让我们回顾一下。
>
> 整个过程就三步：
> 1. **描述** —— 我用中文告诉 Claude 我想要什么
> 2. **生成** —— Claude 写了代码，创建了文件
> 3. **看结果** —— 我在浏览器里打开，看到了成品
>
> 这就是我们整个课程要学的核心循环：**描述 → 生成 → 看结果**。
>
> 后面我们还会加一步：**评估和优化**。比如现在我觉得这个颜色不好看，我可以直接说...

现场演示一次修改：

```
把背景色改成深红色，字体换成更有中国风的感觉。菜名用更大的字。
```

**老师说：**

> 看，这就是第四步：不满意就修改。你只需要说出哪里不对，Claude 就帮你改。
>
> 所以完整的循环是：**描述 → 生成 → 评估 → 优化**。
>
> 这四步，就是接下来 8 次课我们要练到熟练的东西。

#### 课程路线图（10 分钟）

展示 PPT 中的路线图页。简要介绍 8 次课的旅程。

**老师说：**

> 今天是第 1 次课，我们先把工具装好。
>
> 第 2 次课，你们就开始自己动手做东西了。
>
> 第 3-4 次课，我带着你们做一个完整的小工具。
>
> 第 5-6 次课，你们自己选想做什么，我在旁边帮忙。
>
> 第 7-8 次课，你们做自己的项目，最后一天每个人上台展示。
>
> 到第 8 次课结束，你手里会有一个你自己做的、可以用的软件作品。

**要点：**
- 不要在路线图上花太多时间，学员这时候最想做的是"我也想试试"
- 给他们这个期待，然后快速进入安装环节

### 第三部分：终端入门（0:40 - 0:55）

**老师说：**

> 要用 Claude Code，我们需要先认识一个工具：终端。
>
> 终端是什么？简单说，它是你跟电脑对话的另一种方式。平时你用鼠标点来点去，终端是你用打字的方式告诉电脑做什么。
>
> Claude Code 就住在终端里。所以我们先学会打开它、用它。

#### 打开终端

**macOS 学员：**

> 按 `Command + 空格`，输入 `Terminal`（或"终端"），回车。

**Windows 学员：**

> 按 `Windows 键`，输入 `PowerShell`，点击打开。

**助教此时走动，确保每人都打开了终端。**

#### 基础命令（带学员一起做）

投影上逐个演示，每个命令等学员跟上再继续：

| 命令 | 作用 | 老师说 |
|------|------|--------|
| `pwd` | 显示当前位置 | "这告诉你现在在电脑的哪个文件夹里" |
| `ls` | 列出文件 | "这列出当前文件夹里有什么东西"（Windows 用 `dir`） |
| `mkdir test` | 创建文件夹 | "这创建了一个叫 test 的新文件夹" |
| `cd test` | 进入文件夹 | "这进入了 test 文件夹" |
| `cd ..` | 回到上一层 | "这回到上一层" |

**老师说：**

> 就这 5 个命令。整个课程基本上只需要这些。剩下的事情都交给 Claude Code。

**检查点：** "请每个人跑一下 `pwd`，把你看到的结果举手告诉我。" 确保全班跟上。

---

### 休息（0:55 - 1:05）

> 休息 10 分钟。如果你在终端操作遇到问题，休息时间可以找助教帮忙。

### 第四部分：环境安装（1:05 - 1:30）

**老师说：**

> 下半场，我们把 Claude Code 装到你们的电脑上。
>
> 跟着安装指南走，一步一步来。有问题举手，助教会来帮你。

#### 步骤 1：安装 Node.js（10 分钟）

**投影上显示步骤，同时发放打印的安装指南。**

**macOS：**

```bash
# 检查是否已安装
node --version

# 如果没有，用安装包安装
# 打开 USB 中的 node-vXX.X.X.pkg，双击安装
# 或从 https://nodejs.org 下载 LTS 版本

# 安装后重新打开终端，验证
node --version
```

**Windows：**

```powershell
# 检查是否已安装
node --version

# 如果没有，用安装包安装
# 打开 USB 中的 node-vXX.X.X.msi，双击安装
# 安装时勾选 "Add to PATH"

# 安装后重新打开 PowerShell，验证
node --version
```

**老师说：**

> 安装完成的标志：输入 `node --version` 后看到一个版本号，比如 `v22.x.x`。
> 看到版本号的同学举手 —— 很好！
> 还没看到的不要急，助教来帮你。

**常见问题（助教参考）：**

| 问题 | 原因 | 解决 |
|------|------|------|
| `command not found: node` | 终端未刷新 PATH | 关闭终端，重新打开 |
| 安装包无法打开 | macOS 安全设置 | 系统偏好设置 → 安全性 → 允许 |
| "没有权限安装" | 非管理员账户 | 用 USB 中的便携版，或使用网页版备用 |
| 安装极慢 | Wi-Fi 拥堵 | 使用 USB 离线安装包 |

#### 步骤 2：安装 Claude Code（10 分钟）

```bash
# 安装 Claude Code
npm install -g @anthropic-ai/claude-code

# 验证安装
claude --version
```

**老师说：**

> 输入 `claude --version` 看到版本号就说明安装成功了。举手！
>
> 还没成功的 —— 没关系，助教来帮你。

**常见问题（助教参考）：**

| 问题 | 原因 | 解决 |
|------|------|------|
| `EACCES` 权限错误 | npm 全局目录权限 | macOS: `sudo npm install -g @anthropic-ai/claude-code`；Windows: 以管理员身份运行 PowerShell |
| 网络超时 | Wi-Fi 问题 | 使用 USB 中的离线包：`npm install -g ./claude-code-x.x.x.tgz` |
| `npm: command not found` | Node.js 未正确安装 | 回到步骤 1 |

#### 步骤 3：配置 API Key（5 分钟）

**老师说：**

> 最后一步。Claude Code 需要一个"钥匙"才能工作 —— 这叫 API Key。
> 我已经为大家准备好了，不需要你们自己注册。
>
> 请在终端输入以下命令（投影上显示）：

```bash
# 设置 API Key（老师提供具体的 Key）
export ANTHROPIC_API_KEY="sk-ant-xxxxx"    # macOS/Linux
# 或
$env:ANTHROPIC_API_KEY="sk-ant-xxxxx"      # Windows PowerShell
```

**分组分配 Key：**
- 1-5 号学员：Key A
- 6-10 号学员：Key B
- 11-15 号学员：Key C

**老师说：**

> 注意：这个 Key 是课堂专用的，请不要分享给别人。课程结束后我会教你们怎么申请自己的。

## 备用方案汇总

| 场景 | 触发条件 | 应对方案 |
|------|----------|----------|
| API 故障 | 演示时 Claude Code 无响应 | 播放预录屏幕录像 |
| Wi-Fi 崩溃 | 全班无法下载安装包 | USB 离线安装包 |
| 学员电脑无法安装 | 权限限制/系统过旧 | 使用 claude.ai/code 网页版 |
| 演示想法太复杂 | 学员提出需要后端的想法 | 温和引导到预备方案 |
| 安装时间不够 | 超过 50% 学员未完成 | 第 2 次课前 20 分钟继续安装 |
| 学员完全跟不上 | 打不开终端/不会打字 | 助教一对一帮助，先用网页版保证体验 |

## 关键提醒

1. **"哇"时刻是整节课的灵魂。** 如果演示不震撼，后面的安装环节会变成煎熬。花时间准备和练习演示。
2. **安装问题不要恋战。** 一个学员卡超过 5 分钟就切网页版。保护整体节奏比解决个别问题重要。
3. **让学员动手，不要只是看。** 首次交互环节必须每人都做一次，哪怕只是复制粘贴命令。手指敲过键盘和眼睛看过屏幕，记忆效果完全不同。
4. **营造安全感。** 反复告诉学员："做不出来完全正常""没有蠢问题""出错是学习的一部分"。零基础的成年人最怕的是在别人面前显得笨。
5. **不要提前教太多概念。** 今天的目标只有两个：震撼 + 安装。思维模型留给第 2 次课。
