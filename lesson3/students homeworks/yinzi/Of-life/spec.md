# Spec: "生命的意义" — 多维度交互式探索平台

## 一、项目概述

一个纯前端 Web 应用，围绕"生命的意义"这一概念，整合哲学、宗教、科学、文学等多元视角，通过交互式可视化、内容聚合与名人名言，为用户提供沉浸式的探索体验。

**核心定位**：不是给出标准答案，而是展示人类对这一问题的各种思考路径，让用户自己形成理解。

---

## 二、内容维度（知识架构）

以以下七大维度组织所有内容，每个维度下包含代表人物、核心观点、经典文本：

| 维度 | 涵盖范围 |
|---|---|
| **哲学** | 存在主义、荒诞主义、斯多葛主义、虚无主义、儒家、道家、佛教哲学 |
| **宗教与信仰** | 基督教、伊斯兰教、佛教、印度教、犹太教、万物有灵论 |
| **科学** | 进化生物学、宇宙学（人择原理）、神经科学（意识的本质）、物理学（熵与生命） |
| **心理学** | 弗兰克尔意义疗法、马斯洛需求层次、荣格个体化、积极心理学 |
| **文学与艺术** | 但丁、陀思妥耶夫斯基、加缪、托尔斯泰、黑塞等作品中的生命追问 |
| **历史与文化** | 不同文明对生命意义的定义演变（时间线） |
| **当代观点** | TED 演讲、博客、流行哲学、社会调查数据 |

## 三、核心功能模块

### 模块 A：名言瀑布流

- **功能**：以卡片流形式展示来自不同维度的名言，每张卡片包含：
  - 名言原文 + 中文翻译
  - 作者、出处、所属维度
  - 一键复制 / 一键分享
- **交互**：
  - 按维度筛选
  - 随机漫步模式（随机推送一条名言）
  - 收藏/书签功能（localStorage）

### 模块 B：哲学观点关系图（可视化）

- **功能**：交互式力导向图，展示不同哲学流派、思想家之间的影响关系与观点异同
- **节点**：思想家 / 思想流派
- **边**：影响、继承、对立、补充
- **交互**：
  - 拖拽、缩放
  - 点击节点展开详情（生平、核心主张、代表著作）
  - 高亮两节点之间的最短路径（思想溯源）
  - 按维度着色

### 模块 C：时间线（历史演进）

- **功能**：横向可滚动时间轴，展示从古至今关于"生命意义"的关键事件与人物
- **内容**：
  - 轴心时代（公元前 800-200 年）：百家争鸣
  - 中世纪：神学主导
  - 启蒙运动：理性主义转向
  - 19-20 世纪：存在主义爆发
  - 当代：科学与人文的融合
- **交互**：
  - 缩放时间粒度（世纪 / 十年 / 年）
  - 点击时间节点查看详情
  - 筛选特定维度

### 模块 D：外部内容聚合

- **功能**：搜索 & 聚合外部网页资源，展示为统一的内容卡片
- **内容来源**：
  - Wikipedia API（人物/概念摘要）
  - 哲学百科（Stanford Encyclopedia of Philosophy / IEP）
  - YouTube 相关演讲/纪录片（嵌入播放）
  - 学术论文摘要（Google Scholar 公开接口）
- **展示形式**：
  - 卡片列表，含来源标签与可信度标识
  - 内嵌预览（iframe sanitized）
- **注意**：前端直接调用公开 API，不涉及后端代理

### 模块 E："你的意义"（交互式问卷）

- **功能**：引导用户通过一系列选择题反思自己对生命意义的理解
- **流程**：
  1. 用户回答 5-8 道选择题（如"你认为意义来自哪里？" → 选项：自我创造 / 外部赋予 / 不存在 / 关系之中）
  2. 根据答案匹配最接近的哲学观点
  3. 展示匹配结果 + 推荐阅读/观看内容
- **存储**：结果保存到 localStorage，可回溯

### 模块 F："每日一思"（首页每日更新）

- **功能**：首页展示每日更新的名言或观点片段
- **来源**：从名言库中按日期轮换
- **可选扩展**：允许用户订阅每日推送（PWA Notification）

## 五、设计风格方向

- **色调**：深色背景为主（深蓝 / 深灰），象征宇宙与思考的深邃；重点内容使用暖金色或柔和白色作为强调色
- **字体**：中文优先使用思源宋体（标题）+ 思源黑体（正文），营造人文气息
- **氛围关键词**：静谧、深邃、哲思、温暖
- **动画**：柔和淡入、缓慢滚动视差，避免炫目效果
- **响应式**：桌面端为主要体验，移动端保证可读性与基本交互

| 层面 | 选型方向 | 理由 |

| 框架 | React / Vue | 组件化开发，生态丰富 |

| 部署 | GitHub Pages / Vercel | 纯静态站，免费部署 |

## 七、数据模型（概要）

```
Quote {
  id: string

  source: string            // 出处

  id: string
  year: number
  title: string

  id: string
```

## 九、设计原则

1. **多元而非单一**：永远展示多种观点，不暗示某种答案更"正确"
2. **引用而非断言**：所有内容标注出处，让用户自行判断
3. **探索而非灌输**：交互设计引导用户主动发现，而非被动阅读
4. **深度与可访问性平衡**：概念解释从浅入深，允许用户按兴趣深入
5. **离线可用**：核心内容（名言、时间线、关系图数据）为静态数据，不依赖网络
