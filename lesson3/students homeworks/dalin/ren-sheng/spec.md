## 1. 项目定位

### 1.1 一句话定义

一个让普通人从"听过人生算法"升级到"会用人生算法"的交互式学习网页。

### 1.2 产品做什么

不是一篇文章，不是一份 PPT，而是一个 **可点击、可输入、可模拟、可对比** 的学习产品。用户在页面上不仅能"读懂"，还能"试一试"、"算一算"、"对比一下"。

具体形式:
- 系统化讲解"人生算法"的核心概念体系
- 提供 5–8 个交互式小程序，让用户在点击和输入中亲身体会"算法的力量"
- 整合心理学、行为经济学、决策科学、复利思维等多学科视角
- 提供延伸阅读路径，避免成为"信息孤岛"

### 1.3 目标用户画像

| 用户类型 | 特征 | 痛点 | 在本网页获得的价值 |
|---------|------|------|------------------|
| **迷茫的年轻人 (22–30)** | 工作 1–5 年，面临选择焦虑 | 不知如何做长期决策 | 学会用框架代替直觉做选择 |
| **认知升级爱好者** | 读过《穷查理宝典》《纳瓦尔宝典》《原则》 | 概念散落，缺乏系统 | 把零散知识串成可操作系统 |
| **职场中坚 (30–40)** | 资源有限、机会成本高 | 怕走错路，怕错失复利窗口 | 用模拟器验证决策、可视化复利 |
| **学生 / 终身学习者** | 想系统理解决策与思维模型 | 学校不教这些 | 一次性建立"决策工具箱" |

**不适合的用户**: 寻找"成功秘籍"或"快速致富方法"的人。本页面强调"概率思维 + 长期主义",不提供捷径。

### 1.4 用户看完后的具体收获 (可验证)

| 维度 | 看完前 | 看完后 |
|------|-------|--------|
| 概念 | 听过"人生算法",但说不清 | 能用 5 分钟向朋友讲清它的核心和边界 |
| 决策 | 凭感觉做选择 | 知道"决策质量 ≠ 结果质量",能识别决策中的偏差 |
| 复利 | 知道"利滚利" | 能说出复利在能力、关系、资本上的差异 |
| 风险 | 厌恶损失 | 能区分"可承受风险"与"毁灭性风险" |
| 行动 | 立志,但容易放弃 | 拿到一份属于自己的"人生算法清单" |

## 2. 核心概念解释 (内容大纲,不是最终文案)

### 2.1 什么是"人生算法"

**定义** (本网页采用):
> 人生算法 = 一套可重复使用的"输入 → 处理 → 输出"决策模式,让你在面对相似情境时,能以更高质量、更低消耗、更稳定地做出选择。

**关键词拆解**:
- **可重复**: 不是一次性鸡汤,是可复用的"函数"
- **决策模式**: 关注的是"怎么决定",不是"决定什么"
- **更高质量**: 提高期望值,不保证单次结果
- **更稳定**: 减少情绪、偏差、外部噪音的影响

**类比**:
- 不是"地图"(告诉你目的地),而是"导航算法"(告诉你怎么找路)
- 不是"答案",而是"出题方式"
- 不是"模板",而是"编译器"

### 2.2 与"普通建议 / 鸡汤 / 成功学"的区别

| 维度 | 鸡汤 / 成功学 | 人生算法 |
|------|-------------|---------|
| 时间观 | 短期、即时 | 长期、复利 |
| 因果观 | 强调"做了 X 必得 Y" | 强调"概率 + 期望值" |
| 个体性 | 一套话术适合所有人 | 提供框架,由用户填入参数 |
| 失败观 | 失败 = 不努力 | 失败 = 决策与运气共同作用 |
| 可证伪性 | 不可证伪("还不够努力") | 可被反例修正 |
| 信息源 | 个人故事、励志演讲 | 跨学科 (心理学、博弈论、统计) |

> **本网页核心立场**: 警惕"幸存者偏差"和"事后归因"。不教人"如何成功",教人"如何在不确定中持续提高决策质量"。

### 2.3 理解"人生算法"的多个维度

本网页将从 **7 个维度** 切入,每个维度对应一个学习模块 (详见 §3.2):

1. **认知维度** — 你怎么看世界,决定你看到什么
2. **决策维度** — 在不确定中做选择的方法论
3. **复利维度** — 时间的朋友 vs 时间的敌人
4. **选择维度** — 机会成本与可逆性
5. **反馈维度** — 如何从结果中学习而不是被结果欺骗
6. **长期主义维度** — 为什么"慢就是快"
7. **风险维度** — 区分"波动"与"毁灭"

## 3. 页面结构设计

### 3.1 整体信息架构 (IA)

```
首页 (Hero + 概念入口)
├── 模块 1: 认知 — 你眼中的世界
├── 模块 2: 决策 — 在不确定中行动
├── 模块 3: 复利 — 时间的魔法与陷阱
├── 模块 4: 选择 — 机会成本与可逆性
├── 模块 5: 反馈 — 如何向结果学习
├── 模块 6: 长期主义 — 慢就是快
├── 模块 7: 风险 — 不被一击毙命
├── 工具箱 (所有交互小程序集合页)
├── 延伸阅读 (人物 / 书单 / 文章)
└── 我的算法清单 (用户专属页,基于本地存储)
```

### 3.2 首页 (Landing)

**目标**: 30 秒内让用户决定"是否值得停留"。

**展示要素 (从上到下)**:

1. **Hero 区**
   - 主标题: 《人生算法》—— 不是答案,是一种思考方式
   - 副标题: 7 个维度 / 8 个交互模拟器 / 一份可带走的清单
   - CTA 按钮: 「开始 5 分钟体验」+ 「直接看工具箱」

2. **3 段式价值主张**
   - 「鸡汤告诉你 What,本页教你 How」
   - 「不靠故事,靠模拟器」
   - 「带走属于你自己的算法清单」

3. **学习路径预览**
   - 7 个模块的卡片网格,每张卡片:
     - 图标 + 模块名 + 一句话钩子 + 预计时长 (如 "8 分钟")
     - 鼠标悬停展示该模块的核心问题 (例: "你做决策时,在多大程度上是被偏差驱动的?")

4. **底部统计 / 信任锚**
   - "整合自 12 位思想家、5 个学科领域、20+ 经典文献"
   - (可选) 简短引用:芒格 / 纳瓦尔 / Ray Dalio 一句

### 3.3 每个模块的内部结构 (统一模板)

每个模块的页面结构都遵循"**理解 → 体验 → 内化**"三段式:

```
┌────────────────────────────────────────┐
│ 1. 概念引入 (200–400 字)                  │
│    - 一个反直觉问题作为开场              │
│    - 用一个真实案例锚定概念              │
├────────────────────────────────────────┤
│ 2. 核心知识卡片 (3–5 张)                  │
│    - 每张卡片一个子概念                  │
│    - 配图 / 图示                         │
├────────────────────────────────────────┤
│ 3. 交互式小程序 (1–2 个)                  │
│    - 用户输入参数,看到结果              │
│    - 提供"对比模式"                      │
├────────────────────────────────────────┤
│ 4. 名言 / 案例 (1–2 条,精选不堆砌)         │
├────────────────────────────────────────┤
│ 5. 自我反思提示 (1–3 个问题)              │
│    - 用户可输入答案,加入"我的清单"       │
├────────────────────────────────────────┤
│ 6. 延伸阅读 (3–5 条外部链接)              │
├────────────────────────────────────────┤
│ 7. 进入下一模块的引导                     │
└────────────────────────────────────────┘
```

### 3.4 七个模块各自讲什么

| # | 模块 | 核心子概念 | 配套小程序 |
|---|------|---------|-----------|
| 1 | **认知** | 心智模型、第一性原理、认知偏差、贝叶斯更新 | 「偏差自检测试」 |
| 2 | **决策** | 决策质量 vs 结果质量、期望值、可逆 / 不可逆决策 (Bezos 双门模型)、10/10/10 法则 | 「决策成本计算器」 |
| 3 | **复利** | 复利公式、能力 / 关系 / 资本三类复利、复利的负面 (债务、坏习惯) | 「复利成长模拟器」 |
| 4 | **选择** | 机会成本、显性 / 隐性成本、最小遗憾原则、选项漏斗 | 「机会成本对比器」/「人生选择模拟器」 |
| 5 | **反馈** | 结果偏差 (Outcome Bias)、归因谬误、写决策日志、Kahneman 的"慢思考"复盘 | 「决策日志练习」 |
| 6 | **长期主义** | 复利曲线 vs 线性曲线、延迟满足、Lindy Effect、个人 KPI 长期化 | 「长期反馈曲线」 |
| 7 | **风险** | 凯利公式、塔勒布的"反脆弱"、毁灭性风险 vs 波动性风险、风险预算 | 「风险毁灭模拟器」 |

### 3.5 模块之间的学习路径

**默认线性路径** (适合新手):
```
认知 → 决策 → 选择 → 反馈 → 复利 → 长期主义 → 风险 → 我的清单
```
> 顺序逻辑: 先建立"看世界的眼睛"(认知),再学会"在当下行动"(决策、选择),然后"从过去学习"(反馈),最后"为未来布局"(复利、长期主义、风险)。

**进阶跳转路径** (适合老用户):
- 顶部固定导航栏可直达任意模块
- 模块卡片标记"基础 / 进阶 / 高级",老用户可跳过基础
- 提供"按问题进入"的入口:
  - "我面临一个重大选择" → 选择 + 决策
  - "我感觉自己在原地踏步" → 复利 + 反馈
  - "我刚刚做了个错误决定" → 反馈 + 风险

**全局进度**: 顶部 / 侧边栏显示用户已完成的模块 (基于 localStorage,无需登录)。

## 5. 动态程序展示设想 (核心模块)

> 这是本网页与"普通文章页"最大的差异点。每个小程序都不是装饰,而是为了让用户**亲手验证一个反直觉的真理**。

### 5.1 设计原则

1. **每个小程序对应一个"啊哈时刻"** — 不为交互而交互
2. **输入要少,输出要直观** — 5 秒内能看到结果
3. **必须有"对比模式"** — 单一数字无意义,对比才能产生洞察
4. **要有"危险情境"** — 让用户看到"如果我不这样会怎样"

### 5.2 八个核心小程序

#### 5.2.1 复利成长模拟器 (Compound Growth Simulator)

- **所属模块**: 复利
- **用途**: 让用户直观看到"每天 1% 的提升"和"每天 0% 的提升"在 10 年后的差距,以及负复利 (坏习惯) 的代价。
- **输入项**:
  - 起始值 (默认 100)
  - 日 / 年增长率 (滑块: -5% 到 +5%)
  - 时间跨度 (1 年到 30 年)
  - 是否加入"中断" (例:连续 30 天不投入,影响多大)
- **输出**:
  - 主图: 折线图,展示总值随时间变化
  - 对比线: "线性增长 (你以为的)" vs "复利增长 (实际)"
  - 数字看板: 终值、总增长倍数、达到 10x 所需时间
- **教育意义**:
  - 看到"复利曲线的拐点在很晚才出现",理解"为什么大多数人放弃在拐点前"
  - 看到负复利的恐怖 (如 -1% 每天 = 10 年后只剩 2.5%)

#### 5.2.2 人生选择模拟器 (Life Choice Simulator)

- **所属模块**: 选择
- **用途**: 让用户在分叉路口看到不同选择的"分布",而非单一结果。
- **输入项**:
  - 选择 A / B 描述 (例:留在大公司 vs 加入创业公司)
  - 每个选项的预期收益、波动、5 年后失败概率 (用滑块设置)
- **输出**:
  - 双路径轨迹图:展示 A 和 B 在 100 次"平行宇宙"中的结果分布
  - 期望值、最坏 5%、最好 5% 的对比
  - 文字提示: "选 A 的期望收益更高,但选 B 的尾部风险更小"
- **教育意义**:
  - 理解"决策质量 ≠ 单次结果"
  - 学会用"分布"而非"点估计"思考

#### 5.2.3 决策成本计算器 (Decision Cost Calculator)

- **所属模块**: 决策
- **用途**: 帮助用户判断"这个决定值得花多少时间想"。
- **输入项**:
  - 是否可逆? (是 / 否)
  - 决定影响时长 (1 周 / 1 月 / 1 年 / 一生)
  - 决定影响人数 (仅自己 / 家人 / 团队 / 公司)
- **输出**:
  - 决策类型: "Type 1 双向门" 或 "Type 2 单向门" (Bezos 模型)
  - 建议投入决策时间: "10 分钟" / "1 天" / "1 周" / "请咨询多人"
  - 警告: 当用户为"可逆小事"投入过多时间,提示"过度决策"
- **教育意义**:
  - 大部分人对小事过度纠结,对大事草率决定
  - 学会"决策预算"

#### 5.2.4 长期反馈曲线 (Long-term Feedback Curve)

- **所属模块**: 长期主义
- **用途**: 揭示"大多数有价值的事,反馈都是延迟的"。
- **输入项**:
  - 选择活动: 健身 / 学习一门技能 / 写作 / 投资 / 建立关系
  - 每天投入时长
  - 时间跨度
- **输出**:
  - 双曲线对比:"主观感受 (努力 vs 即时回报)" vs "客观成果"
  - 标记"放弃高发期" (通常在 3 个月、1 年节点)
  - 终点输出"如果坚持完整周期的可能成果"
- **教育意义**:
  - 看到"努力与回报的曲线不重合",理解为什么坚持难
  - 知道"放弃高发期"在哪,提前防备

#### 5.2.5 机会成本对比器 (Opportunity Cost Comparator)

- **所属模块**: 选择
- **用途**: 把"看不见的成本"可视化。
- **输入项**:
  - 当前选项 (例:打 1 小时游戏)
  - 替代选项 (例:学习、运动、社交)
  - 时间频率 (每天 / 每周)
- **输出**:
  - 1 年后、5 年后的"复利化机会成本"
  - 双柱状图对比:你选择的 A vs 你放弃的 B
  - 文字提示: "你不是在'放松 1 小时',你是在用 5 年后的某个能力交换"
- **教育意义**:
  - 机会成本不是"另一个选择的价值",而是"另一个选择经过复利后的价值"

#### 5.2.6 风险毁灭模拟器 (Ruin Risk Simulator)

- **所属模块**: 风险
- **用途**: 演示"哪怕只有 1% 的毁灭性风险,长期下来也会让你出局"。
- **输入项**:
  - 单次胜率 (滑块)
  - 单次盈亏比
  - 是否使用"全押" / 凯利仓位 / 半凯利
  - 玩多少局
- **输出**:
  - 100 条蒙特卡洛路径,展示资金曲线
  - 出局率 (Ruin Probability)
  - 关键洞察: "胜率 60%,盈亏比 2:1,看起来很赚 — 但全押 100 局后出局率 X%"
- **教育意义**:
  - 期望值为正 ≠ 长期生存
  - 仓位管理比胜率更重要

#### 5.2.7 偏差自检测试 (Cognitive Bias Self-test)

- **所属模块**: 认知
- **用途**: 用经典心理学题目让用户体验自己的偏差。
- **题目示例 (5–10 道)**:
  - 林达问题 (合取谬误)
  - 框架效应 (同样数据,正面 / 负面表述)
  - 锚定效应 (随机数字影响估计)
  - 损失厌恶 (确定收益 vs 概率收益)
- **输出**:
  - 偏差雷达图,展示用户在每类偏差上的"中招度"
  - 个性化提示: "你在锚定偏差上较敏感,做谈判 / 报价时尤其要警惕"
- **教育意义**:
  - 直接体验"我以为我理性,其实不是"
  - 知道哪些偏差是自己的"高发病种"

#### 5.2.8 决策日志练习 (Decision Journal)

- **所属模块**: 反馈
- **用途**: 引导用户记录一次真实决策,3 个月后回看。
- **输入项**:
  - 你正在做的决定是什么?
  - 你预测的最可能结果是什么? (% 概率)
  - 你的依据是什么?
  - 哪些信息你其实不知道?
- **输出**:
  - 一份结构化的决策日志 (可导出 Markdown)
  - 设置提醒 (基于浏览器通知): "3 个月后回来复盘"
- **教育意义**:
  - 区分"决策质量"与"运气"
  - 长期下来,看到自己的真实判断准确率,而不是事后归因

### 5.3 小程序之间的关联

- 完成 5.2.1 (复利) 的用户,系统建议进入 5.2.5 (机会成本) — 因为机会成本本质是"复利化"的
- 完成 5.2.6 (风险) 的用户,系统建议进入 5.2.3 (决策成本) — 因为风险高的决策应该花更多时间想
- 所有小程序的输入参数都可以"保存到我的清单",形成个性化档案

### 7.2 名言放置原则

**避免堆砌的三条铁律**:

1. **一个概念配一句即可** — 多了用户会跳过
2. **名言必须服务于上下文,不是装饰** — 引用前后必须有"为什么这句话与这里相关"的解释
3. **优先引用"反直觉"的话,而非"正确但平庸"的话**
   - ❌ "成功来自坚持" (正确但无信息量)
   - ✅ "如果你想说服别人,诉诸利益,而非诉诸理性。" — 芒格 (反直觉,引发思考)

### 7.3 名言模块的视觉设计

每条名言卡片包含:
- 引文 (大号字)
- 头像 + 姓名 + 身份
- 出处 (书 / 演讲 / 年份)
- 一句"编辑导读" — 告诉用户"这句话在本模块语境下意味着什么"
- (可选) "应用到我的清单"按钮

### 7.4 案例库 (非名言,而是真实情境)

每个模块至少配一个案例,形式为"问题—决策—结果—反思":

- **决策模块案例**: Bezos 创立 AWS — 一个"双向门"看似的"单向门"决定
- **风险模块案例**: LTCM (长期资本管理公司) — 高期望值 + 毁灭性风险的经典案例
- **复利模块案例**: 芒格在书单上的复利 (每天读书 vs 不读)
- **反馈模块案例**: Ray Dalio 的"决策机器"是如何在 1982 年大败后建立的

## 8. 视觉与交互设计

### 8.1 整体风格

**设计基调**: **冷静、克制、有学术感,但不古板**。

**关键词**: 极简 / 留白 / 高对比 / 数据可视化美学 / 一点点科技感

**参考产品**:
- Stripe Press (网页版书籍,排版极佳)
- Our World in Data (数据可视化的克制美感)
- Wait But Why (插画 + 长文,但本网页插画更克制)
- Apple Human Interface Guidelines 文档站

### 8.2 配色方案

| 角色 | 浅色模式 | 深色模式 | 用途 |
|------|---------|---------|------|
| 背景 | `#FAFAF7` (米白) | `#0F1115` (深蓝黑) | 页面底色 |
| 主文字 | `#1A1A1A` | `#E8E8E8` | 正文 |
| 次文字 | `#6B7280` | `#9CA3AF` | 注释、出处 |
| 强调色 | `#C2410C` (赭石) | `#FB923C` (暖橙) | CTA、关键概念 |
| 数据色 1 | `#0F766E` (深青) | `#2DD4BF` | 折线图主线 |
| 数据色 2 | `#7C3AED` (紫) | `#A78BFA` | 折线图对比线 |
| 警示色 | `#DC2626` | `#EF4444` | 风险 / 错误 |
| 卡片背景 | `#FFFFFF` | `#1A1F2B` | 概念卡片 |

**理由**: 米白 + 赭石给"思想类内容"以纸质书的稳重感,避免常见科技网站的"冷蓝"造成的距离感。

### 8.3 字体

- **中文标题**: 思源宋体 (Noto Serif SC) — 体现"经典 / 思想"
- **中文正文**: 思源黑体 (Noto Sans SC)
- **英文 / 数字**: Inter (正文) + JetBrains Mono (代码 / 数字)
- **引言 / 名言**: 思源宋体 + 斜体效果 (慎用)

### 8.4 信息层级 (Visual Hierarchy)

```
H1 (模块标题)        — 48px, Serif, 加粗
H2 (子概念)          — 32px, Serif, 中粗
H3 (卡片标题)        — 22px, Sans, 加粗
正文                — 17px, Sans, 行高 1.7
注释 / 引用出处       — 14px, Sans, 灰色
名言                — 28px, Serif, 斜体
代码 / 公式           — 16px, Mono
```

**留白原则**: 段落间距 ≥ 1.5em;模块之间使用大块留白 (≥ 80px) 而非分隔线。

### 8.5 展示方式 (各种"载体")

| 载体 | 用途 | 何时使用 |
|------|------|---------|
| **概念卡片** | 包装一个子概念 | 模块知识区 |
| **对比卡片** | 左右对比 (鸡汤 vs 算法) | §2.2 类内容 |
| **时间线** | 展示思想史演进 | 首页 / 思想家专题 |
| **流程图** | 决策路径、双向门 | 决策模块 |
| **折线图 / 柱状图** | 复利、机会成本 | 模拟器内部 |
| **散点图 / 蒙特卡洛** | 风险路径 | 风险模拟器 |
| **雷达图** | 多维度自评 | 偏差自检 |
| **节点图** | 概念关系 | 模块结尾的"知识图谱" |
| **引用卡** | 名言 + 出处 + 导读 | 模块尾部 |
| **手写注释** (装饰风格) | 偶尔点缀,营造"思考笔记"感 | 关键转折处 |

### 8.6 让"深度内容"不枯燥的 7 个技巧

1. **反直觉开场**: 每个模块开头用一个"违反常识"的问题或现象
2. **可点击就点击**: 关键词、数据、名言都可点击,提供更多内容
3. **进度反馈**: 每完成一个模块,有"+1 智慧"的微动画 (但克制,不游戏化过度)
4. **微互动**: 滚动到特定位置时,图表自动播放动画 (一次,不重复)
5. **故事化**: 抽象概念用具体场景代入 (不是"假设你有 100 万",而是"假设小李 25 岁,月薪 1.5 万")
6. **节奏控制**: 每 3 段文字插入一个交互 / 图表 / 引言,避免长文字疲劳
7. **意外感**: 在"我的清单"页给用户一个汇总图 (例:"你今天加了 5 条反思,你属于'反思型'用户")

### 8.7 动画原则

- 出现动画: 慢 (300–500ms),缓动 (ease-out)
- 不使用任何"营销页"风格的炫技动画
- 图表入场: 数据从 0 到目标值的过渡 (Animate-on-scroll)
- 暗黑模式切换: 平滑过渡而非突变
- 严格遵守 `prefers-reduced-motion` 用户偏好

## 10. 技术实现建议

> 本节仅为技术选型方向,**不写具体代码**。

### 10.1 总体架构

- **类型**: 静态站点 (Static Site) + 客户端交互
- **理由**:
  - 无需后端 (用户数据全部存 localStorage)
  - 部署简单 (Vercel / Netlify / GitHub Pages)
  - 加载快,SEO 友好
  - 隐私友好 (无服务器 = 无数据收集)

### 10.2 推荐技术栈

| 层 | 技术 | 理由 |
|---|------|------|
| 框架 | **Next.js (App Router)** 或 **Astro** | Astro 更适合内容站,Next.js 适合复杂交互 |
| UI 语言 | **React + TypeScript** | 生态丰富,小程序组件可复用 |
| 样式 | **Tailwind CSS** + CSS Variables | 快速迭代,易做深色模式 |
| 内容管理 | **MDX** | Markdown 中可直接嵌入 React 组件,完美贴合"正文 + 小程序"模式 |
| 图表 | **Recharts** (基础) + **D3.js** (高级) | 复利图、雷达图用 Recharts;蒙特卡洛、决策树用 D3 |
| 动画 | **Framer Motion** | 入场动画 + 微互动,API 友好 |
| 状态管理 | **Zustand** 或 React Context | 轻量,管理"我的清单"等全局状态 |
| 持久化 | **localStorage** + 一层封装 | 用户数据本地化 |
| 字体 | Google Fonts (Noto Serif SC, Inter) + 自托管 | 隐私 + 性能 |
| 部署 | **Vercel** | 与 Next.js 一键集成 |
| 分析 | **Plausible** 或 **Umami** (隐私友好) | 不用 GA |

### 10.3 性能与可访问性

- Lighthouse 目标: Performance ≥ 90, Accessibility ≥ 95
- 图表懒加载 (Intersection Observer)
- 关键 CSS 内联,字体异步加载
- 全键盘可达,ARIA 标签完整
- 颜色对比度 ≥ AA 级
- 响应 `prefers-reduced-motion`、`prefers-color-scheme`

### 10.4 响应式

- 移动端优先设计 (375px 起)
- 断点: 375 / 768 / 1024 / 1440
- 模拟器在移动端简化交互 (滑块代替拖拽)
- 长文章在移动端单栏,PC 端可选侧边栏目录

### 10.5 内容工作流

- 每个模块为一个 `.mdx` 文件
- 小程序作为独立 React 组件,可在任意 MDX 中插入
- 内容修改不需要工程师介入
- (未来扩展) 接入 CMS (如 Sanity / Contentlayer),便于多人协作

### 10.6 国际化 (未来)

- 默认中文 (zh-CN)
- 架构上预留 i18n,未来可扩英文版
- MDX 文件按语言目录组织: `content/zh/`, `content/en/`

### 10.7 不使用的技术 (反向清单)

明确不使用,避免将来踩坑:
- ❌ 任何后端服务 (本期无服务器)
- ❌ 用户登录系统 (隐私优先)
- ❌ Google Analytics / Facebook Pixel (隐私)
- ❌ 任何 AI 聊天机器人 (本期不引入,避免分散学习注意力)
- ❌ 重型 UI 库 (如 Ant Design,与极简风格不符)

## 13. 风险与边界

### 13.1 内容风险

- **避免成为另一锅鸡汤**: 严格遵守 §2.2 的对比表,内容审稿时用此表逐条对照
- **避免精英主义**: 不假设读者已经财务自由 / 知识渊博,案例参数贴近普通工薪
- **避免过度简化**: 在"简化"和"准确"之间倾向准确,必要时增加"想深入"折叠区

### 13.2 设计风险

- **避免过度交互**: 不为交互而交互,每个互动都要服务于一个"啊哈时刻"
- **避免数据焦虑**: 模拟器不应让用户产生"我已经输了"的绝望感,应给出"现在开始也不晚"的可视化路径

### 13.3 技术风险

- **localStorage 容量**: 超过 5MB 时降级提醒用户导出
- **图表性能**: 蒙特卡洛 100 条路径在低端手机的渲染优化
- **SEO**: 大量 MDX 内容需要预渲染,注意构建时间

### 13.4 明确不做的事

- ❌ 不做"测一测你的人生算法等级"这类娱乐化测试
- ❌ 不做"扫码获取完整版"这类增长 hack

## 14. 附录

### 14.1 关键术语表 (将在网页内提供完整版)

- **算法 (Algorithm)**: 一组明确、可重复执行的步骤,本网页扩展为"决策模式"
- **复利 (Compounding)**: 收益再投资带来的非线性增长
- **机会成本 (Opportunity Cost)**: 选择一项时放弃的次优项的价值
- **期望值 (Expected Value)**: 各结果概率加权后的平均值
- **凯利公式 (Kelly Criterion)**: 在已知胜率与赔率下,使长期资金增长率最大的下注比例
- **黑天鹅 (Black Swan)**: 罕见、极端、事后被合理化的事件
- **反脆弱 (Antifragile)**: 在波动和压力中受益的特性,而不仅仅是抵抗
- **林迪效应 (Lindy Effect)**: 非生命物 (思想、技术) 的预期剩余寿命与已存在时间成正比
- **Type 1 / Type 2 决策**: Bezos 提出的不可逆 / 可逆决策分类
