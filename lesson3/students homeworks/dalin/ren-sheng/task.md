# 项目任务拆解

本文档将《人生算法》交互式学习网页拆分为 **4 个开发阶段**:

- **阶段一**: 项目基础搭建 — 工程地基、设计系统、内容管线
- **阶段二**: MVP 核心交付 — 首页 + 7 模块静态内容 + 4 个核心小程序
- **阶段三**: 完整功能扩展 — 剩余 4 个小程序 + 思想家专题 + 知识图谱
- **阶段四**: 体验打磨与发布 — 性能、可访问性、每日一练、上线

每阶段内的任务按依赖关系排序,后置任务不应在前置任务完成前启动。

## 已决策问题记录

> 原"待确认问题"区,10 项已于 2026-05-10 全部决策完毕。下方记录每条决策结论与影响范围。
> 后续若决策需要调整,请在此追加新版本,不要直接覆盖旧条目。

### Q1. 内容撰写主体 ✅ 已决策 (2026-05-10)
- **决策**: AI 起草 + 人工审核。AI 负责按 spec.md §3.3 七段式模板产出初稿,人工负责事实核对、反鸡汤审稿 (按 §2.2 对比表)、案例真实性核查。
- **影响**: T2.2.1–T2.2.7 执行模式确定。每个模块预计 AI 起草 1 天 + 人工审稿 1 天。

### Q2. 视觉资产来源 ✅ 已决策 (2026-05-10)
- **图标**: 统一使用 **Lucide** (MIT 协议,风格克制贴合 §8.1 极简基调)。禁止混用其他图标库。
- **头像 (思想家 / 名言卡片)**: 使用 **抽象首字母圆形** + spec.md §8.2 的配色 token (例:芒格 = "CM" 配赭石底)。**不使用真实人像**,规避肖像权与版权风险。
- **手写注释装饰**: 用 SVG 自绘最简符号 (箭头、下划波浪),不引入第三方手写素材。
- **影响**: T1.2.2、T2.1.2、T3.2.1 直接采用此规范。

### Q3. MDX 工具选型 ✅ 已决策 (2026-05-10)
- **决策**: 使用 **Velite** (https://velite.js.org/) 作为内容层 + `@next/mdx` 渲染。
- **理由**: Contentlayer 已停止维护;Velite 活跃维护、TypeScript 原生、Zod schema 校验、构建产物轻。
- **影响**: T1.1.3 按此实施。`content/zh/**/*.mdx` 由 Velite 在构建期校验 + 类型生成。

### Q4. 风险模拟器免责声明 ✅ 已决策 (2026-05-10)
- **决策**: 凯利 / 蒙特卡洛类模拟器 (T2.3.1 复利、T3.1.1 人生选择、T3.1.3 风险毁灭) 在结果区底部一律显示固定免责文案:
  > "本模拟器仅用于教育目的,不构成投资、理财或其他专业建议。任何决策请结合自身情况并咨询专业人士。"
- **同步动作**: 建议 spec.md §13.4 在下次更新时增加此条。
- **影响**: T2.3.1、T3.1.1、T3.1.3 的"完成标准"加入此项。

### Q5. 决策日志的提醒兜底 ✅ 已决策 (2026-05-10)
- **决策**: 决策日志同时提供 **两条提醒路径**,默认提示用户至少选其一:
  - (推荐) 导出 `.ics` 日历事件,加入用户自己的日历 (Apple / Google / Outlook)。
  - 浏览器通知 (作为补充,不作为唯一手段)。
- **理由**: 浏览器通知失效场景太多 (清缓存、换设备、关闭权限),`.ics` 是跨平台、长期可靠的兜底。
- **影响**: T3.1.4 的"预期产出"增加 `.ics` 生成逻辑。

### Q6. 每日一练题库可持续性 ✅ 已决策 (2026-05-10)
- **决策**: 当前阶段 (M1–M3) **不实现每日一练**。该功能原属阶段四,已暂缓。
- **后续**: 阶段四规划时,题库目标提至 ≥ 50 条 (覆盖 7 周以上),并预留 CMS 接入接口。
- **影响**: 阶段三任务保持现状;阶段四相关任务 (T4.1.1–T4.1.3) 推迟。

### Q7. 多端同步方案 ✅ 已决策 (2026-05-10)
- **决策**: 提供 **JSON 导入 / 导出对** 作为多端同步的兜底,不引入服务器。
  - 导出: 一键将 localStorage 全量数据 (反思、模拟器历史、测试结果等) 序列化为 `.json` 下载。
  - 导入: 读取 `.json` 文件,合并或覆盖 (用户选择) 当前 localStorage 数据。
- **影响**: T2.4.2 范围扩大,从"仅 Markdown 导出"扩展到"Markdown 导出 + JSON 导入导出对"。建议在阶段二实施时同步完成 JSON 导出,导入功能可推到阶段三。
